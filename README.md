# Screen Watcher — تطبيق مراقبة وقراءة الشاشة

تطبيق Android (Kotlin) يقرأ قيمة رقمية من منطقة تحددها على الشاشة، يقارنها بقيمة مستهدفة،
وينقر تلقائياً على نقطة "التالي" إذا لم تتطابق. يتوقف فوراً عند التطابق.

## كيف يعمل

1. **AccessibilityService** يقرأ شجرة عناصر الشاشة الفعلية (لا screenshot/OCR) — أدق وأسرع.
2. **Overlay شفاف** يسمح لك بسحب مستطيل لتحديد منطقة القراءة، ونقطة واحدة لتحديد مكان النقر.
3. **حلقة فحص** كل ~1.2 ثانية: يقرأ → يقارن → ينقر إذا لم يطابق → يكرر.

## فتح المشروع وبناءه

1. ثبّت **Android Studio** (أحدث إصدار).
2. افتح المجلد `TikTokWatcher` كمشروع Android Studio (Open existing project).
3. اتركه يحمّل Gradle (أول مرة قد تأخذ وقت لتنزيل التبعيات).
4. وصّل جوالك (فعّل USB debugging) أو استخدم محاكي Android 7+ (API 24+).
5. اضغط Run ▶ لتثبيت التطبيق.

> ملاحظة: المحاكي العادي يكفي للتجربة، لكن لتجربة فعلية مع تيك توك يفضل جهاز حقيقي.

## خطوات الاستخدام داخل التطبيق

1. افتح التطبيق → اضغط **"فتح إعدادات Accessibility"** → فعّل خدمة "Screen Watcher Service".
2. اضغط **"تحديد منطقة القراءة"** → سيطلب صلاحية "العرض فوق التطبيقات" أول مرة (وافق) →
   بعدها يظهر overlay، اذهب لتيك توك وحدد بالسحب المنطقة اللي فيها القيمة (مثلاً عدد المشاهدات).
3. اضغط **"تحديد نقطة النقر"** → حدد بنقرة واحدة مكان زر "التالي" (سحب لأعلى أو زر السهم).
4. اكتب **القيمة المستهدفة** ونوع المقارنة (أكبر/أصغر/يساوي).
5. اضغط **"حفظ الإعدادات"**.
6. اضغط **"ابدأ المراقبة"** → التطبيق يبدأ يفحص ويقرر تلقائياً.
7. عند التطابق: إشعار + توقف تلقائي.

## بنية الكود

| ملف | الوظيفة |
|---|---|
| `MainActivity.kt` | الواجهة: صلاحيات، تحديد المنطقة/النقطة، حفظ، تشغيل/إيقاف |
| `OverlayPickerService.kt` | نافذة شفافة فوق كل التطبيقات لتحديد مستطيل أو نقطة بالسحب/النقر |
| `ScreenWatcherService.kt` | AccessibilityService: القراءة من الشجرة، استخراج الرقم، المقارنة، النقر (gesture) |
| `WatcherConfig.kt` | تخزين الإعدادات في SharedPreferences (بدون أي بيانات شخصية) |

## نقاط مهمة / قيود معروفة

- **القراءة تعتمد على AccessibilityNodeInfo**، يعني تحتاج يكون النص "ظاهر" كعنصر نص حقيقي
  وليس مرسوماً داخل Canvas/Video مباشرة. تيك توك عادة يعرض الأرقام كـ TextView عادي فوق الفيديو،
  فهذا يشتغل تمام في أغلب الحالات. لو لقيت إنه ما يقرأ شي، جرب توسّع منطقة التحديد شوي.
- **استخراج الرقم** يدعم صيغ مثل `12.3K`, `1,200`, `1.2M` تلقائياً.
- **النقر فعلي عبر `dispatchGesture`**، يعني هو نقرة حقيقية على الشاشة (تحاكي لمسة المستخدم).
- التطبيق **لا يحفظ** أي بيانات تجمعها أو سجل، فقط يقرأ/يقارن/ينقر لحظياً كما طلبت.
- تأكد ما تحدد منطقة القراءة فوق نفس مكان نقطة النقر، حتى ما يصير تعارض.

## تعديلات سريعة محتملة

- تغيير `checkIntervalMs` في `ScreenWatcherService.kt` لتسريع/تبطيء الفحص.
- إذا تريد "التالي" يكون سحبة (swipe) بدل نقرة بسيطة (مثلاً سحب فوق فيديو تيك توك)،
  أقدر أعدّل `performTap` إلى `performSwipeUp` بسهولة — قلّي وأضيفها.
